# K-Quant Deck: 한국투자 Open API 기반 국내주식 자동매매 시스템

K-Quant Deck은 한국투자증권 Open API를 사용해 국내주식 후보 선별, 현재가 조회, 주문 실행, 포지션 추적, 손절/익절 감시, 웹 대시보드를 제공하는 Python 기반 자동매매 프로젝트입니다.

이 프로젝트의 목적은 수익률을 과장하는 것이 아니라, 실거래 자동화 시스템을 유지보수 가능한 구조로 만들고 운영 안정성, 상태 저장, 로그 확인, 검증 가능성을 높이는 것입니다.

## 핵심 기능

- 한국투자 Open API 실계좌/모의계좌 인증
- 국내주식 현재가 조회
- 자동 후보 선별
- 예산 기반 매수 수량 계산
- 실제 주문 실행
- 포지션 상태 저장
- 같은 종목 중복 매수 방지
- 실패 종목 자동 제외
- 손절/익절 조건 감시
- Flask 기반 웹 대시보드
- 학습 가능한 거래 데이터 누적 구조

## 실행 구조

1. `.env`에서 API 키, 시크릿, 계좌번호를 불러옵니다.
2. 한국투자 Open API 토큰을 발급하거나 캐시를 사용합니다.
3. 자동 후보 선별 조건에 맞는 국내 종목을 찾습니다.
4. 현재가, 등락률, 거래량, 실패 이력, 보유 상태를 확인합니다.
5. 예산과 종목당 한도를 기준으로 주문 가능 수량을 계산합니다.
6. 매수/매도 조건이 맞으면 주문을 실행합니다.
7. 포지션, 실패 종목, 학습 데이터, 로그 상태를 저장합니다.
8. 웹 대시보드에서 실행 상태와 실시간 로그를 확인합니다.

## 설치

Python 설치 후 프로젝트 폴더에서 한 번만 실행합니다.

```powershell
install_dependencies.bat
```

또는 직접 설치할 수도 있습니다.

```powershell
python -m pip install flask requests
```

## 설정

`.env.example` 파일을 복사해서 `.env` 파일을 만들고 본인 값으로 채웁니다.

```env
KIS_APP_KEY=your_app_key_here
KIS_APP_SECRET=your_app_secret_here
KIS_ACCOUNT=00000000-00
KIS_MODE=real
```

주의: `.env`는 실제 비밀정보 파일입니다. 절대 GitHub, 블로그, 포트폴리오 ZIP에 넣지 마세요.

## 자동매매 직접 실행 예시

```powershell
python kis_autotrade.py --auto-pick --budget 50000 --per-symbol-budget 10000 --max-qty 10 --poll-seconds 10 --stop-loss-percent 10 --take-profit-percent 0 --execute
```

`--execute`를 붙이면 실제 주문이 나갈 수 있습니다. 테스트할 때는 먼저 `--execute` 없이 실행해서 로그를 확인하세요.

## 전략 개요

현재 전략은 규칙 기반 자동매매입니다. AI가 수익을 보장하는 구조가 아니라, 거래 과정에서 생기는 데이터를 저장하고 이후 점수화/학습 구조로 확장할 수 있도록 설계되어 있습니다.

자동 후보 선별은 가격, 등락률, 거래량, 보유 여부, 실패 이력, 종목명 필터 등을 기준으로 후보를 추립니다. 매수 후에는 저장된 포지션을 기준으로 손절/익절 조건을 감시합니다.

## 리스크 관리

- 총 예산 제한
- 종목당 투자금 제한
- 종목당 최대 수량 제한
- 같은 종목 중복 매수 방지
- 실패 종목 쿨다운
- 손절 조건
- 주문 전 주문 가능 수량 확인

실거래 시스템은 항상 손실 가능성이 있습니다. 이 프로젝트는 자동화 구조를 만드는 것이며, 투자 판단과 책임은 사용자에게 있습니다.

## 상태 파일

런타임 상태는 `.cache/` 폴더에 저장됩니다. 이 폴더는 개인 실행 기록과 토큰이 들어갈 수 있으므로 배포에서 제외합니다.

- `position_state.json`: 추적 중인 보유 포지션
- `symbol_failures.json`: 실패 종목 제외 상태
- `order_lock.json`: 중복 주문 방지 상태
- `trade_training_data.jsonl`: 거래/학습 데이터
- `trade_model_state.json`: 학습 모델 상태
- `kis_token_real.json`: 한국투자 접근 토큰

## 보안 주의사항

배포하거나 포트폴리오로 제출할 때 다음 파일은 반드시 제외합니다.

- `.env`
- `흠흠.env`
- `흠흠.txt`
- `.cache/`
- `kis_token_real.json`
- 실제 계좌번호, API 키, API 시크릿, 접근 토큰이 들어간 모든 파일

공개용에는 `.env.example`만 포함합니다.

## 배포용 ZIP 만들기

PowerShell에서 실행합니다.

```powershell
powershell -ExecutionPolicy Bypass -File .\make_release.ps1
```

생성된 `release/` 폴더의 ZIP 파일을 공유하기 전에 `.env`, `.cache`가 들어있지 않은지 한 번 더 확인하세요.

## 현재 한계

- 실거래 데이터가 충분히 쌓이기 전까지 학습 모델의 판단력은 제한적입니다.
- 백테스트와 실거래 환경은 완전히 같지 않습니다.
- API 장애, 장중 급변동, 주문 지연, 체결 차이로 결과가 달라질 수 있습니다.
